### 2.2 数据完整性问题

**当前问题**:

1. **合同金额 ≠ 实际收入**
   - 合同金额是计划收入，但没有实际开票和收款记录
   - 无法区分：合同签订、已开票、未开票、已收款

2. **发票数据不完整**
   - 发票类型（收款/付款）缺少与项目的直接关联
   - 发票状态（待开票/已开票/已收票）缺失

3. **项目进度计算粗糙**
   - 仅基于任务完成数，没有权重
   - 没有里程碑概念

4. **缺少时间维度分析**
   - 没有创建时间、更新时间追踪（虽有字段但未充分利用）
   - 无法做趋势分析

## 三、需要增加的数据表

### 3.1 核心数据补充

```sql
-- 1. 项目里程碑
CREATE TABLE milestones (
  id INTEGER PRIMARY KEY,
  project_id INTEGER,
  name TEXT,              -- 里程碑名称
  due_date DATE,         -- 计划日期
  actual_date DATE,      -- 实际完成日期
  status TEXT,            -- 待完成/已完成/已延期
  created_at DATETIME
);

-- 2. 任务依赖关系
CREATE TABLE task_dependencies (
  id INTEGER PRIMARY KEY,
  task_id INTEGER,        -- 当前任务
  depends_on_id INTEGER,   -- 依赖的任务
  dependency_type TEXT,    -- blocks/blocked by/relates to
  created_at DATETIME
);

-- 3. 项目收入记录（替代仅靠合同金额）
CREATE TABLE project_income (
  id INTEGER PRIMARY KEY,
  project_id INTEGER,
  invoice_id INTEGER,      -- 关联发票
  amount REAL,             -- 收入金额
  expected_date DATE,      -- 预计收款日期
  actual_date DATE,        -- 实际收款日期
  status TEXT,             -- 待收款/已收款/逾期
  created_at DATETIME
);

-- 4. 项目支出记录（替代仅靠管理费用）
CREATE TABLE project_expense (
  id INTEGER PRIMARY KEY,
  project_id INTEGER,
  category TEXT,           -- 人工/材料/分包/管理/其他
  amount REAL,
  expense_date DATE,
  description TEXT,
  receipt_no TEXT,
  created_at DATETIME
);

-- 5. 客户/甲方信息
CREATE TABLE clients (
  id INTEGER PRIMARY KEY,
  name TEXT NOT NULL,
  contact_person TEXT,
  phone TEXT,
  email TEXT,
  address TEXT,
  created_at DATETIME
);

-- 6. 风险登记
CREATE TABLE risks (
  id INTEGER PRIMARY KEY,
  project_id INTEGER,
  title TEXT,
  description TEXT,
  probability TEXT,        -- 高/中/低
  impact TEXT,             -- 高/中/低
  mitigation TEXT,         -- 应对措施
  status TEXT,             -- 识别/处理中/已解决/已接受
  created_at DATETIME
);

-- 7. 问题登记
CREATE TABLE issues (
  id INTEGER PRIMARY KEY,
  project_id INTEGER,
  title TEXT,
  description TEXT,
  priority TEXT,            -- 紧急/高/中/低
  status TEXT,              -- 打开/处理中/已解决/关闭
  assignee_id INTEGER,       -- 负责人
  due_date DATE,
  created_at DATETIME
);

-- 8. 项目文档
CREATE TABLE documents (
  id INTEGER PRIMARY KEY,
  project_id INTEGER,
  name TEXT,
  file_path TEXT,          -- 文件路径
  file_type TEXT,           -- pdf/doc/xlsx等
  category TEXT,            -- 合同/报告/图纸/其他
  created_at DATETIME
);

-- 9. 活动日志
CREATE TABLE activity_log (
  id INTEGER PRIMARY KEY,
  project_id INTEGER,
  entity_type TEXT,         -- project/contract/task等
  entity_id INTEGER,
  action TEXT,              -- created/updated/deleted
  changes TEXT,            -- JSON格式的变更内容
  actor TEXT,               -- 操作人
  created_at DATETIME
);
```

### 3.2 现有表需要补充的字段

**projects 表增强**:
- `client_id` - 关联客户
- `description` - 项目描述
- `progress` - 手动设置的进度百分比
- `priority` - 优先级（紧急/高/中/低）
- `budget_warning_threshold` - 预算预警阈值（可配置）

**contracts 表增强**:
- `payment_status` - 付款状态（未付款/部分付款/已付清）
- `payment_terms` - 付款条款

**invoices 表增强**:
- `invoice_type` - 发票类型（增值税专用票/普通票）
- `tax_rate` - 税率
- `tax_amount` - 税额
- `status` - 发票状态（待开/已开/已收/已作废）

**tasks 表增强**:
- `priority` - 优先级
- `estimated_hours` - 预估工时
- `actual_hours` - 实际工时
- `parent_task_id` - 父任务ID（支持子任务）
- `labels` - 标签（JSON数组）

## 四、流程优化建议

### 4.1 项目全生命周期流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        项目立项                                  │
│  创建项目 → 关联客户 → 制定预算 → 设置里程碑 → 分配团队         │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                        合同管理                                  │
│  签订合同 → 合同分类(销售/采购) → 付款条款 → 执行跟踪          │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                        发票与收款                               │
│  开具发票 → 收款确认 → 收入记录 → 发票状态追踪                  │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                        项目执行                                  │
│  任务分解 → 任务分配 → 执行跟踪 → 里程碑达成 → 风险监控        │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                        成本管理                                  │
│  预算编制 → 费用报销 → 成本统计 → 利润分析 → 预警提示           │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                        项目验收                                  │
│  验收准备 → 验收执行 → 问题整改 → 验收确认 → 项目归档           │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 核心审批流程

1. **合同审批流程**
   - 拟定 → 审核 → 批准 → 签订

2. **发票开具流程**
   - 申请 → 审核 → 开具 → 送达

3. **付款审批流程**
   - 申请 → 复核 → 批准 → 支付

4. **验收确认流程**
   - 申请 → 初验 → 整改 → 终验 → 确认

## 五、界面优化建议

### 5.1 当前界面问题

1. **信息密度低**
   - 项目列表没有显示关键指标（预算/进度/状态）
   - 每个面板只显示单一类型数据

2. **视觉层次不清晰**
   - 状态标签没有明显的视觉区分
   - 金额没有颜色编码（收入vs支出）

3. **缺少数据可视化**
   - 没有图表展示项目状态
   - 没有进度指示器

4. **交互效率低**
   - 列表项点击后才能看到详情
   - 没有批量操作
   - 没有快捷搜索/过滤

## 七、总结

**优势**:
- 3D地球可视化展示项目分布（独特亮点）
- 基本的项目管理结构完整
- 预算和费用管理已有基础
- 城市数据支持地理位置

**需要改进**:
- 数据完整性：缺少客户、收入记录、实际支出
- 流程完整性：缺少审批流、任务依赖
- 界面体验：信息密度低、缺少可视化
- 财务追踪：合同金额≠实际收入

**建议路径**:
1. 先完善数据模型（Phase 1）
2. 优化核心用户体验（Phase 2-3）
3. 逐步增加高级功能（Phase 4）
